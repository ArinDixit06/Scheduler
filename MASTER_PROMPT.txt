SECTION 3 — BACKEND API (Express + Node.js)
--------------------------------------------------------------------------------

Generate the complete Express API with the following route groups. Every route
must include: Zod request validation, JWT auth middleware, error handling, and
JSDoc comments.

  3A. AUTH ROUTES  /api/auth
      POST   /register          — email + password registration
      POST   /login             — returns accessToken + refreshToken
      POST   /refresh           — rotate refresh token
      POST   /logout            — invalidate refresh token
      POST   /oauth/google      — Google OAuth callback
      POST   /oauth/apple       — Apple OAuth callback
      POST   /oauth/microsoft   — Microsoft OAuth callback
      POST   /forgot-password   — send reset email
      POST   /reset-password    — validate token + update password

  3B. USER ROUTES  /api/user
      GET    /me                — current user profile
      PATCH  /me                — update profile, timezone, working hours
      PATCH  /me/energy-hours   — update peak energy window
      POST   /me/energy-log     — log today's energy score (1-5)
      GET    /me/energy-log     — get energy log (last 30 days)
      DELETE /me                — account deletion (GDPR)

  3C. TASK ROUTES  /api/tasks
      GET    /                  — list with filters: status, project, tag, due
      POST   /                  — create task (triggers AI scoring)
      GET    /:id               — single task + subtasks + comments
      PATCH  /:id               — update any field
      DELETE /:id               — soft delete
      POST   /:id/complete      — mark done + log actual time
      POST   /:id/subtasks      — create subtask
      GET    /:id/dependencies  — get dependency graph
      POST   /:id/dependencies  — add dependency
      POST   /:id/attachments   — upload file to S3
      GET    /:id/comments      — list comments
      POST   /:id/comments      — add comment
      GET    /ai/next-action    — AI: what should I work on RIGHT NOW?
      POST   /ai/bulk-schedule  — AI: schedule unscheduled tasks this week

  3D. PROJECT ROUTES  /api/projects
      GET    /                  — list user projects
      POST   /                  — create project
      GET    /:id               — project + tasks + milestones + members
      PATCH  /:id               — update project
      DELETE /:id               — archive project
      GET    /:id/progress      — completion stats + burndown data
      POST   /:id/members       — invite member
      DELETE /:id/members/:uid  — remove member

  3E. CALENDAR ROUTES  /api/calendar
      GET    /events            — events in date range (all sources merged)
      POST   /events            — create internal event
      PATCH  /events/:id        — update event
      DELETE /events/:id        — delete event
      POST   /sync/google       — trigger manual Google Calendar sync
      POST   /sync/outlook      — trigger manual Outlook sync
      GET    /events/:id/prep   — get or generate AI meeting prep brief
      POST   /events/:id/transcript — upload meeting audio for transcription
      GET    /availability      — get free/busy slots in a range
      POST   /smart-schedule    — AI: find best slot for a new event/task

  3F. HABIT ROUTES  /api/habits
      GET    /                  — list habits
      POST   /                  — create habit
      PATCH  /:id               — update habit
      DELETE /:id               — delete habit
      POST   /:id/log           — log entry for today
      GET    /:id/history       — log history (last 90 days)
      GET    /streaks           — all habits with current streak

  3G. FOCUS ROUTES  /api/focus
      POST   /sessions          — start a focus session
      PATCH  /sessions/:id      — update (pause, resume, end)
      GET    /sessions          — history with stats
      GET    /sessions/stats    — aggregate stats: total hours, avg session
      GET    /ambient-sounds    — list available ambient sound options

  3H. INTEGRATION ROUTES  /api/integrations
      GET    /                  — list connected integrations + status
      POST   /connect/:provider — initiate OAuth for a provider
      DELETE /:id               — disconnect integration
      POST   /:id/sync          — force sync
      GET    /:id/status        — last sync time, error info

  3I. NOTIFICATION ROUTES  /api/notifications
      GET    /                  — list notifications (unread first)
      POST   /read/:id          — mark single as read
      POST   /read-all          — mark all as read
      GET    /settings          — get notification preferences
      PATCH  /settings          — update notification preferences
      POST   /device-token      — register FCM device token

  3J. AI ROUTES  /api/ai
      POST   /chat              — conversational AI copilot (streaming SSE)
      GET    /insights          — get current AI insights for user
      POST   /insights/:id/dismiss — dismiss an insight
      GET    /weekly-recap      — generate or return this week's recap
      POST   /email-draft       — draft email reply from context
      POST   /task-breakdown    — break a large task into subtasks
      GET    /overload-check    — analyze if user is overloaded this week

  3K. SEARCH ROUTES  /api/search
      GET    /universal         — search tasks, events, notes, habits at once
      GET    /suggestions       — autocomplete suggestions

  3L. ANALYTICS ROUTES  /api/analytics
      GET    /time-by-project   — hours logged per project (last 30d)
      GET    /task-completion   — completion rate over time
      GET    /focus-quality     — focus session trends
      GET    /energy-vs-output  — energy score vs tasks completed correlation

SECTION 4 — AI SERVICE (packages/ai-service)
--------------------------------------------------------------------------------

Generate a standalone AI service module using the Anthropic Claude API. Every
function must accept a userId so it can pull personalized context from the DB
before calling Claude. Use streaming where the response is conversational.

  4A. CopilotChat(userId, messages[], stream: boolean)
      System prompt includes: user's name, timezone, today's tasks, today's
      events, top 3 priorities, current energy score.
      Returns: streaming SSE response or full text.
      Example intents to handle: "what should I do now", "clear my afternoon",
      "reschedule my 3pm", "draft a reply to [person]", "am I on track this week"

  4B. ScoreTaskPriority(task: TaskInput, userContext: UserContext)
      Scores 0-100 based on: deadline urgency, estimated effort, project
      importance, user's stated priorities, time available today.
      Returns: { score: number, reasoning: string, suggestedTimeBlock: string }

  4C. GenerateMeetingPrepBrief(event: CalendarEvent, userId)
      Pulls last 3 email threads with attendees, last meeting notes with them,
      open action items related to the meeting topic.
      Returns: { agenda: string[], context: string, actionItems: string[],
                 suggestedQuestions: string[] }

  4D. SummarizeMeetingTranscript(transcript: string, attendees: string[])
      Returns: { summary: string, actionItems: ActionItem[], decisions: string[],
                 nextMeetingDate?: string }

  4E. DetectOverload(userId, weekStart: Date)
      Analyses: total estimated task minutes vs available work minutes,
      meeting density, current energy trend.
      Returns: { overloaded: boolean, severity: LOW|MEDIUM|HIGH,
                 tasksToDefer: string[], message: string }

  4F. GenerateWeeklyRecap(userId, weekEnd: Date)
      Returns: { tasksCompleted: number, focusHours: number, topWin: string,
                 habitStreaks: object, energyAvg: number,
                 nextWeekSuggestions: string[], message: string }

  4G. SmartSchedule(userId, item: { title, durationMinutes, deadline? })
      Fetches calendar free slots, energy peaks, existing task blocks.
      Returns: { suggestedSlots: TimeSlot[], reasoning: string }

  4H. BreakdownTask(task: Task)
      Returns: { subtasks: { title, estimatedMinutes }[], totalMinutes: number }

  4I. DraftEmailReply(emailContext: { subject, from, body }, userContext)
      Returns: { draft: string, tone: FORMAL|CASUAL, confidence: number }

  4J. GenerateProactiveInsights(userId)
      Run nightly via BullMQ cron. Checks: overdue tasks, habit streak at risk,
      upcoming deadlines, context-switch heavy days, positive momentum.
      Writes AIInsight records to DB. Triggers push notification if high priority.
